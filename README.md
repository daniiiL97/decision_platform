# Decision Platform

Проект выполнен в рамках курса «Интеллектуальные платформы и автоматизация принятия решений».

Репозиторий содержит контур принятия решений для предварительной оценки прочности бетонной смеси. Технолог задаёт состав смеси и возраст образца. Модель рассчитывает ожидаемую прочность на сжатие. Затем результат сравнивается с технологическими границами, после чего приложение предлагает дальнейшее действие и сохраняет расчёт в журнал.

В качестве данных используется [Concrete Compressive Strength Dataset](https://archive.ics.uci.edu/dataset/165/concrete+compressive+strength) из UCI Machine Learning Repository.

## Disclaimer

В проекте используются реальные лабораторные данные из открытого набора UCI.

Требуемая прочность и запас для автоматической рекомендации задаются пользователем в приложении.

Модель обучена с помощью [mljar-supervised](https://supervised.mljar.com/).

Для запуска приложения требуется распакованная модель в папке `models`.

Приложение предназначено для учебной демонстрации и предварительной оценки. Оно не заменяет пробный замес и лабораторное испытание.

## Getting Started

Требования

* Python 3.10, 3.11 или 3.12
* pip
* распакованная модель в папке `models`

```bash
git clone https://github.com/daniiiL97/decision_platform.git
cd decision_platform

python -m venv .venv
.venv\Scripts\activate
source .venv/bin/activate

pip install -r requirements.txt
streamlit run app.py
```

После запуска приложение доступно на локальном порте `8501`.

Если модель отсутствует, её нужно обучить в ноутбуке `train_automl_concrete_strength.ipynb`, скачать архив и распаковать в папку

```text
models/
    AutoML_Concrete_Strength/
    config.json
```

## Usage

В приложении есть три раздела.

| Раздел               | Содержание                                                         |
| -------------------- | ------------------------------------------------------------------ |
| Оценка состава смеси | ввод состава, запуск расчёта, просмотр зон решения и шагов контура |
| Как работает система | данные, метрики модели, объяснение результата и ограничения        |
| Журнал решений       | история расчётов за последние 24 часа и обратная связь             |

В основном разделе доступны типовые составы и учебные сценарии.

```text
Показать красную зону
Показать пограничный случай
Показать зелёную зону
Показать ошибку ввода
```

После выбора сценария значения можно изменить вручную и повторно запустить расчёт.

## How It Works

Контур состоит из 8 шагов.

| Шаг | Что происходит                                                |
| --- | ------------------------------------------------------------- |
| 1   | Проверка введённых значений                                   |
| 2   | Проверка физически невозможных значений и диапазонов обучения |
| 3   | Формирование входного вектора из восьми параметров            |
| 4   | Расчёт прогнозной прочности ML моделью                        |
| 5   | Расчёт 90% интервала и сравнение с технологическими границами |
| 6   | Выбор действия по положению интервала                         |
| 7   | Показ прогноза, зон решения и основания для рекомендации      |
| 8   | Запись результата и обратной связи в журнал                   |

Модель возвращает прогноз прочности на сжатие в МПа. Дополнительно рассчитывается 90% интервал неопределённости.

Для решения используются две границы.

| Граница              | Назначение                                              |
| -------------------- | ------------------------------------------------------- |
| Требуемая прочность  | минимальная прочность, которую должен обеспечить состав |
| Граница рекомендации | требуемая прочность с учётом запаса                     |

Зоны решения

| Зона    | Что это означает                                           |
| ------- | ---------------------------------------------------------- |
| Красная | весь расчётный диапазон находится ниже требуемой прочности |
| Жёлтая  | диапазон пересекает требование или границу рекомендации    |
| Зелёная | весь диапазон находится выше требования и заданного запаса |

Если в составе нет воды, слишком мало цемента или возраст образца меньше суток, расчёт модели не выполняется.

Если параметр выходит за диапазон обучения, прогноз сохраняется, но приложение показывает предупреждение. Такой результат нужно трактовать осторожнее.

## Model

Используется датасет [Concrete Compressive Strength Dataset](https://archive.ics.uci.edu/dataset/165/concrete+compressive+strength).

В модель входят восемь параметров.

* Cement
* Blast Furnace Slag
* Fly Ash
* Water
* Superplasticizer
* Coarse Aggregate
* Fine Aggregate
* Age

Целевая переменная

```text
Concrete Compressive Strength
```

Обучение выполнено через [mljar-supervised](https://github.com/mljar/mljar-supervised) в режиме `Compete`.

Тип задачи

```text
Regression
```

Метрики на тестовой выборке

| Метрика                | Значение |
| ---------------------- | -------- |
| MAE                    | 2.91 МПа |
| RMSE                   | 4.09 МПа |
| R²                     | 0.936    |
| Покрытие 90% интервала | 91.0%    |

Радиус интервала рассчитан по ошибкам модели на validation выборке. Он используется, чтобы показать не только центральный прогноз, но и диапазон возможной прочности.

## Project Structure

```text
decision_platform/
├── app.py
├── requirements.txt
├── train_automl_concrete_strength.ipynb
├── models/
│   ├── AutoML_Concrete_Strength/
│   └── config.json
└── logs/
    └── decisions.jsonl
```

Основные функции в `app.py`

| Функция                     | Назначение                                                    |
| --------------------------- | ------------------------------------------------------------- |
| `step1_validate`            | проверка входных данных                                       |
| `step2_checks`              | проверка физически невозможных значений и диапазонов обучения |
| `step4_predict`             | вызов модели                                                  |
| `step5_compare_with_policy` | расчёт интервала и определение зоны решения                   |
| `step6_choose_action`       | выбор дальнейшего действия                                    |
| `step8_log`                 | запись результата в журнал                                    |
| `update_feedback`           | сохранение обратной связи                                     |
| `find_scenario_composition` | подбор учебного сценария через текущую модель                 |
| `render_zone_visual`        | отображение зон решения и прогнозного интервала               |
| `render_pipeline_trace`     | показ расчётов по восьми шагам                                |
| `run_pipeline`              | запуск контура принятия решений                               |
